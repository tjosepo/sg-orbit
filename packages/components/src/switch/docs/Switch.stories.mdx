# Switch

<Tagline>A switch offers an immediate activate or deactivate choice to a user.</Tagline>

<ComponentInfo
    usage={"import { Switch } from \"@sharegate/orbit-ui\";"}
    ariaPath="#checkbox"
    githubPath="/packages/components/src/switch/src"

## Best practices

- Use to trigger an immediate action such as an activation/deactivation.

## Accessibility

- Text should be short and descriptive.
- Text should be descriptive and complete enough to be understood without relying on an icon. Using an icon to carry information will render it non-accessible.
- Do not change the label based on the state of the switch. The label should represent the "on/checked" state of the switch.

## Related components

- If the objective is to select elements from a list, the [checkbox](?path=/docs/checkbox--default-story) or the [radio button]() should be used instead.
- To provide contextual help, use a [tooltip](?path=/docs/tooltip--default-story).
- To trigger an action, use a [button](?path=/docs/button--default-story).

## Usage

### Default

A switch can be either checked or unchecked and should always show the current state of the object.

<Preview>
    <Story name="default">
        <Switch>Engines</Switch>
    </Story>
</Preview>

### Checked

A switch can be checked.

<Preview>
    <Story name="checked">
        <Switch defaultChecked>Engines</Switch>
    </Story>
</Preview>

### No label

A switch can be rendered without a label. When rendered without a label a title should be close to the switch making it clear what happens when the toggle is turned on and off.

<Preview>
    <Story name="no label">
        <Switch aria-label="Engines" />
    </Story>
</Preview>

### Disabled

A switch can be disabled. A disabled switch should continue to display it's current state(checked or unchecked).

<Preview>
    <Story name="disabled">
        <Switch disabled>Engines</Switch>
    </Story>
</Preview>

### Icon

A switch can have an [icon](?path=/docs/icon-gallery--page) after its text.

<Preview>
    <Story name="icon">
        <Switch>
            <Text>Pressurization</Text>
            <LightbulbIcon />
        </Switch>
    </Story>
</Preview>

### Icon with tooltip

A switch can have an [icon](?path=/docs/icon-gallery--page) with a [tooltip](?path=/docs/tooltip--default-story) after its label.

<Preview>
    <Story name="tooltip">
        <Switch>
            <Text>Pressurization</Text>
            <TooltipTrigger slot="icon">
                <LightbulbIcon />
                <Tooltip>Earth is a small town with many neighborhoods in a very big universe.</Tooltip>
            </TooltipTrigger>
        </Switch>
    </Story>
</Preview>

### Counter

A switch can have a [counter](?path=/docs/counter--default-story) after its text.

<Preview>
    <Story name="counter">
        <Switch>
            <Text>Engines</Text>
            <Counter>6</Counter>
        </Switch>
    </Story>
</Preview>

### Size

A Switch can be used in small version to accommodate smaller spaces, such as a column row. When used in small format, the switch should not contain an icon.

<Preview>
    <Story name="size">
        <Inline alignY="center">
            <Switch size="sm">Engines</Switch>
            <Switch>Engines</Switch>
        </Inline>
    </Story>
</Preview>

### Reverse

A switch toggle can be positioned to the left or to the right of its label.

<Preview>
    <Story name="reverse">
        <Switch reverse>
            <InfoIcon />
            <Text>Engines</Text>
            <Counter variant="divider">6</Counter>
        </Switch>
    </Story>
</Preview>

### Validation

A switch can display a validation state to communicate to the user whether the current value is valid or invalid. Implement your own validation logic in your app and pass either `"valid"` or `"invalid"` to the input via the `validationState` prop.

The example below illustrates how one would validate if the user has checked the switch.
